# Verificação do release 2.0.0

Data da campanha: 19/07/2026 (America/São_Paulo).

Este documento separa evidência automatizada, inspeção em navegador e validação física. Números
de Chromium headless não são apresentados como desempenho de hardware real.

## 7A — Balanceamento final

Comando:

```powershell
npx vitest run src/game/simulation/BalanceBattery.test.ts src/game/mechanics/serve.test.ts --reporter=verbose
```

Resultado: **20/20 testes aprovados**.

| Gate | Resultado | Faixa aprovada |
|---|---:|---:|
| Rallies / seeds | 1.000 / 20 | ≥ 1.000 / ≥ 20 |
| Mediana de contatos | 6 | 4–8 |
| Pontos decisivos | 78,4% | ≥ 65% |
| Maior share de zona | 42,0% | ≤ 45% |
| Partidas / seeds | 30 / 10 | 30 / 10 |
| Mediana de duração | 10,63 min | 8–12 min |
| p90 de duração | 13,50 min | ≤ 15 min |

A realização física do saque permaneceu idêntica nas três dificuldades com os mesmos draws; a
dificuldade continua variando percepção, decisão e consistência técnica. Todas as faixas ficaram
dentro do design aprovado, portanto a decisão de release é **não fazer retuning especulativo**.

## 7B — Performance e compatibilidade

Matriz executada sobre o mesmo `dist/` com Chromium, Firefox, WebKit, Pixel 5/Chromium e iPhone
12/WebKit emulados: **5/5 aprovados**, sem `console.error`, `pageerror` ou warning de
`PCFSoftShadowMap`. O WebKit Playwright para Windows não expõe Web Audio; o fallback silencioso
foi validado e impede que a ausência da API derrube o jogo.

| Perfil forçado | Draw calls máximos | Triângulos máximos | Orçamento |
|---|---:|---:|---:|
| Desktop alto — Chromium | 231 | 146.662 | 250 / 500.000 |
| Desktop alto — Firefox | 227 | 147.954 | 250 / 500.000 |
| Desktop alto — WebKit | 232 | 145.704 | 250 / 500.000 |
| Mobile médio — Android emulado | 172 | 103.020 | 180 / 250.000 |
| Mobile médio — iPhone emulado | 167 | 101.382 | 180 / 250.000 |

O mobile caiu de 230 para 172 draw calls ao substituir 60 passes de sombra das submalhas das
atletas por duas sombras blob instanciadas (uma por time). O desktop alto preserva as sombras
dinâmicas; tier baixo e touch usam blobs. O `?tier=` de diagnóstico agora congela o tier pedido,
evitando que a medição mude de perfil no meio da amostra.

Baseline Chromium headless alto: 229,4 draw calls médios pelo `renderer.info`, máximo 233,
149.195 triângulos médios e heap do browser sem crescimento na janela curta. O FPS headless com
WebGL por software (6,3 fps; p95 183,4 ms) é apenas regressão relativa, não evidência de hardware.
O JS principal mede 221,43 kB gzip (limite 250 kB) e todo o `dist/` mede 0,818 MiB sem compressão
(limite de payload comprimido 10 MiB).

Decisão de pós-processamento: permanece desligado. MSAA nativo e os presets atuais preservam a
leitura; não há ganho visual demonstrado que justifique adicionar composer, bundle e custo por
frame nesta release.

Playtest visual em navegador real: desktop 1280×720 no tier alto e touch 844×390 no tier médio
iniciaram partidas sem erro visual. O touch mediu 172 draw calls, 103.092 triângulos e zero
atletas projetando sombra; as blobs mantiveram o contato visual com a quadra. Em 390×844, a pausa
de orientação e o menu permaneceram legíveis e operáveis. A navegação corrente terminou sem erros
ou avisos de console.
