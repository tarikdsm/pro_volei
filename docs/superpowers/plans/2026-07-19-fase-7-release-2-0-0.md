# Fase 7 — Polimento e release 2.0.0 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Status:** 7A–7C concluídas localmente em 19/07/2026; 7D (metadados, gates e publicação) ativa.

**Goal:** Fechar balanceamento, compatibilidade, performance, resiliência, auditoria e publicação
do Pró Volei como release 2.0.0 rastreável por versão e SHA.

**Architecture:** A simulação já aprovada permanece intacta salvo evidência estatística contrária.
Gates de release ficam em testes/configs e as evidências em `docs/release/`; metadados de build são
injetados pelo Vite e apresentados pela UI. Recuperação de plataforma fica fora de `game/`, sem
introduzir DOM/WebGL na simulação.

**Tech Stack:** TypeScript strict, Three.js r185, Vite 8, Vitest 4, Playwright 1.61, GitHub Actions
e GitHub Pages.

## Global Constraints

- Fluxo `main`-only, sem branch/PR/amend/force-push; commits pequenos e push apenas após gates.
- Zero URLs remotas no runtime e bundle JS inicial de no máximo 250 kB gzip.
- Draw calls ≤ 180 no mobile médio e ≤ 250 no desktop alto; triângulos ≤ 250 mil/500 mil.
- Física, RNG e regras não mudam por dificuldade, acessibilidade, cosmético ou FPS.
- `npm run check`, build, E2E dev/prod/offline, matriz de browsers, performance e playtest reais
  devem estar verdes antes da tag.
- Medição física em Android intermediário e iPhone, ou equivalentes documentados, é gate humano;
  emulação nunca deve ser descrita como medição física.
